The Text Editor

IMAGE - UPDATE SCREENSHOT ONCE CODE COMPLETION IS IN - USE DEFAULT SETTINGS FOR TEXT SIZE ETC.

The Text Editor in GameMaker is used to edit various text files. You will mostly use this to edit Object Events and Scripts, however it is also used for:

  • Shaders
  • Timelines
  • Notes
  • Extension files
  • Creation code of a room, and the creation code of a room instance

If your game contains any Included Files of the formats given below, they will be editable in the Text Editor:

Included Files Supported Formats

  • JSON – .json
  • YAML – .yml
  • INI – .ini, .conf
  • XML – .xml, .xsd, .tld, .jsp, .pt, .cpt, .dtml, .rss, .opml
  • CSV – .csv
  • Markdown – .txt, .md 
  • C, C++ – .c, .cpp, .cxx, .cc, .c++, .h, .hh, .hpp, .hxx, .h++
  • Objective-C, Objective-C++ – .m, .mm, .h, .hh
  • Java – .java, .bsh

Enable Text Editor

The Text Editor is currently opt-in, and is enabled through the Text Editor 2 Preferences.

After enabling the Text Editor, please restart your IDE. Opening any of the assets listed above should then open the new Text Editor.

Editor Frame

The Text Editor always opens in a new workspace tab, containing the editor itself, with a frame around it:

IMAGE - screenshot showing the "editor frame" and the "editor"

This page covers all parts of the Editor Frame. For information on writing code inside the editor, see Editing Code.

Workspace Tab Menu

 Right-clicking the workspace tab for the Text Editor will display the following options:

IMAGE - OPTIONS

These are the Text Editor-specific options:

  • Show in Asset Browser: Selects the asset being edited in The Asset Browser
  • Show in Inspector: Shows the asset being edited in the The Inspector
  • Open in Explorer: Shows the asset being edited in your operating system's file browser
  • Copy Full Path: Copies the full path of the asset being edited to your clipboard

Navigation Bar

This is the most important part of the Editor Frame.

Here you can change which asset you are working on, the document within that asset being edited, and the part of that document that is focused.

IMAGE - NAVIGATION

This bar contains the following parts:

  • Orientation: The bar first contains a drop-down menu to detach the Navigation Bar into its own draggable window, or attach it to any of the four sides of the frame. Clicking this button will cycle through each option, or you click on the arrow to select the option you need.
    • Changing this will also update its corresponding preference, meaning any future Text Editor windows will use this new orientation. Inversely, changing the preference will update the orientation used in editor windows.
  • Asset Selection: The first drop-down lets you change the asset being edited. This can be any of the assets listed at the top of the page. You can click the  button to add a new asset from a list of supported assets.

    ALT/OPT + 1 opens this dropdown, and ALT/OPT + SHIFT + 1 opens the "Create Asset" menu.
  • Document Selection: The second drop-down lets you change the document being edited. This is most useful in objects, as each event is a different document, you can change the event in focus. You can click the  button to add a new document (which, in case of an object, adds a new event).

    ALT/OPT + 2 opens this dropdown, and ALT/OPT + SHIFT + 2 opens the "Create Document" menu.
  • Declaration Selection: The third drop-down lets you jump to a declaration within the open document, for example, a variable or a function being declared.

    ALT/OPT + 3 opens this dropdown.
  • Splitter: The final, right-most button on this bar is the Splitter. Clicking and dragging this will let you split the Text Editor into a duplicate view, allowing you to edit multiple documents, or different positions in the same document, on one screen.

IMAGE - SPLITTER IN ACTION

Gutter

The gutter is a vertical column that appears to the left of your code, where the line numbers are displayed:

IMAGE - GUTTER

The gutter is further divided into different "rails", which are simply columns inside the gutter:

  • Togglables: The left-most gutter is highlighted and contains bookmarks and breakpoints. When editing a GML document, clicking on this rail for a line of code will create a breakpoint.
  • Contextual: Moving right, the contextual rail will show the Quick Fixes button where a syntax error appears.
  • Line Numbers: The right-most rail within the gutter shows line numbers for the open document.
  • Folding: Folding buttons appear on the edge of the gutter, allowing you to fold documents and {} bracket blocks within a document.
    • Additionally, switch cases and multiline comments are also foldable.

Vertical Scrollbar

The rightmost edge of the Editor Frame displays a vertical scrollbar, which has some additional functionality beyond just scrolling.

IMAGE - SCROLLBAR FEATURES

The scrollback allows overscroll, so that the end of the document can be edited at a higher vertical level on the screen. It will also display diagnostics (i.e. errors, warnings and suggestions) as small lines:

IMAGE - DIAGNOSTICS

You can enable a minimap next to the scrollbar in the Text Editor 2 Preferences. This displays a preview of your code, to give you a general idea of the code structure, so you can just between different parts of the document.

Hovering over a part of the minimap will temporarily display the code at that position:

IMAGE - HOVER MINIMAP

The preview will continue to be displayed until your mouse cursor leaves the minimap area.

Horizontal Scrollbar

The horizontal scrollbar appears at the bottom of the frame, to scroll left or right. This has the following parts:

IMAGE - HORIZONTAL SCROLLBAR PARTS

The left-most drop-down menu controls the zoom level of all Text Editors in the current session. This does not persist across sessions, as such it is only meant for temporary use during the current IDE session. For changing the size of your text permanently, do so through the Text Editor 2 Preferences or modify the DPI in the General Preferences.

To the right of the scrollbar itself, the current line and column number of the text cursor are displayed. Clicking on this opens the "Go To Line" dialog (which can also be opened with / + G):

IMAGE - CLICK TO OPEN GO TO LINE

Next you have the overwrite mode display, this shows either INS (the default typing behaviour) or OVR (overwriting existing text), which is toggled with the Insert key on your keyboard.

Finally you have a drop-down to change the language of the document. When switching between GML Code and GML Visual, the focused event will be modified.

IMAGE - CHANGE LANGUAGE

The languages that appear in the list will depend on the type of document that is open.
